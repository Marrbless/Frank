# Autonomous Improvement Matrix

Date: 2026-04-30

This matrix extends the total repo assessment into a backlog that a future autonomous pass can use without rediscovering the whole repo. It is intentionally broad, but each row is phrased as a bounded slice. It does not authorize rewrites, schema migrations, dependency additions, destructive cleanup, or behavior breaks unless the row explicitly says to stop for human input.

## How To Use

1. Pick exactly one row unless the rows are explicitly independent.
2. Read the files named in the row before editing.
3. Add or update the smallest executable check that proves the intended behavior.
4. Keep public behavior and durable schemas compatible unless the row says human input is required.
5. Run the listed validator, then the broader gate when the row touches shared behavior.
6. Update the row status only when the implementation and validation evidence are complete.

Status values:

- `candidate`: ready for future selection.
- `blocked`: needs human input, live-device evidence, or external credentials.
- `done`: implemented and validated.

Effort values:

- `S`: one focused edit or doc pass.
- `M`: one implementation slice with tests.
- `L`: several related files or a shared behavior surface.
- `XL`: needs decomposition before implementation.
